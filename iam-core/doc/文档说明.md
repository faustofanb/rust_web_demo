## 📖 文档更新指南

1. **文档规范**
   - 所有文档使用 Markdown 格式
   - 文档命名使用中文，清晰明了
   - 代码示例需要标注语言类型

2. **文档分类**
   - 核心文档：架构设计、系统设计等基础文档
   - 开发文档：开发指南、API 文档等实操性文档
   - 运维文档：部署、监控、日志等运维相关文档

3. **文档维护**
   - 定期更新文档内容
   - 保持版本一致性
   - 及时补充新功能文档

4. **文档评审**
   - 重要文档需要团队评审
   - 保持文档的准确性
   - 确保文档的可读性迎来到 IAM Core 系统文档！这里包含了系统的完整文档。
